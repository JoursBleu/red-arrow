# Red Arrow：一个 Android SSH 隧道小工具，随时随地访问你的内网服务

## 背景：你是否遇到过这些场景？

在家部署了一套 **Ollama / vLLM / LocalAI**，跑着 Llama、Qwen 或者 DeepSeek 模型，局域网内用得很爽。结果出门之后，手机上想调用一下自己的模型——连不上。

或者你在公司内网有一台开发机，跑着 Jupyter Notebook、数据库、Web 服务，回到家想继续干活——连不上。

又或者家里的 NAS、Home Assistant、监控摄像头，出门在外想看一眼——还是连不上。

**问题的本质很简单：你的服务在 NAT 后面，没有公网 IP，外面访问不到。**

解决方案其实也很成熟：**SSH 隧道**。只要你有一台带公网 IP 的服务器（哪怕是最便宜的 VPS），就能通过 SSH 把内网服务"带出来"。

但是在手机上操作 SSH 隧道一直不太方便。要么用 Termux 手敲命令，要么用各种 SSH 客户端配端口转发，步骤繁琐且容易断。

所以我写了 **Red Arrow** —— 一个极简的 Android SSH 隧道工具，专门解决这个问题。

---

## Red Arrow 是什么？

一句话：**在 Android 手机上一键建立 SSH 隧道，把远程内网服务映射到本地代理端口。**

它的工作方式：

```
你的手机 (Red Arrow)
    ↓ SSH 加密隧道
你的公网 VPS（跳板机）
    ↓ 内网可达
你的内网服务（LLM / NAS / 开发机 ...）
```

连接建立后，手机上会启动 SOCKS5（默认 :1080）和 HTTP（默认 :8080）本地代理。你只需要把需要访问内网的应用设置代理指向 `127.0.0.1:1080`，流量就会通过 SSH 隧道到达内网。

**而且它监听 0.0.0.0**，意味着同一 Wi-Fi 下的其他设备（笔记本、iPad）也可以用你手机的 IP 作为代理地址，共享这条隧道。

## 典型使用场景

### 场景一：远程调用家里的 LLM

家里的 3090 跑着 Ollama，监听 `192.168.1.100:11434`。

1. 买一台最便宜的云服务器（2 块钱/月那种就行），确保你家里的机器能 SSH 到这台服务器
2. 在手机上打开 Red Arrow，填入云服务器的 SSH 信息，点连接
3. 手机上的 AI 客户端（比如 Open WebUI、ChatBox）设置代理为 `127.0.0.1:1080`
4. 直接访问 `http://192.168.1.100:11434` —— 流量通过 SSH 隧道到达家里的内网

**效果：出门在外也能用自己的本地大模型，不用把模型暴露到公网。**

### 场景二：在家访问公司开发环境

公司有 GitLab、Jenkins、内部 Wiki，都在内网 `10.0.x.x` 段。

同样的思路：公司内网一台机器能 SSH 到你的跳板机，手机连上跳板机，流量就能到达公司内网。

### 场景三：手机做"随身代理网关"

手机连上 SSH 隧道后，因为监听 `0.0.0.0`，你的笔记本、平板都可以把代理指向手机 IP。

比如在咖啡厅，笔记本设置 HTTP 代理为 `手机IP:8080`，就能通过手机的 SSH 隧道访问远端内网服务。相当于手机变成了一个便携的加密代理网关。

## 功能一览

| 功能 | 说明 |
|---|---|
| SSH 隧道 | 通过 SSH 建立加密通道，支持密码和密钥认证 |
| SOCKS5 代理 | 本地 SOCKS5 代理，默认端口 1080 |
| HTTP 代理 | 本地 HTTP 代理，默认端口 8080 |
| 监听 0.0.0.0 | 局域网内其他设备可共享隧道 |
| 代理鉴权 | 可选用户名密码，防止他人蹭用 |
| 密钥管理 | 内置生成 Ed25519/RSA 密钥对，支持发送公钥到服务器 |
| 断线重连 | SSH 断开后自动重连，指数退避策略 |
| 连接健康检查 | 每 10 秒检测 SSH 会话，异常自动恢复 |
| 流量统计 | 实时显示上传/下载字节数 |
| 活跃连接 | 按 IP 分组显示当前代理连接 |
| 前台服务 | WakeLock 保活，隧道长时间稳定运行 |
| Material Design 3 | 浅色/深色/跟随系统主题 |
| 中英双语 | 界面支持中文和英文 |

## 为什么不用 frp / ZeroTier / Tailscale？

这些都是好工具，但各有取舍：

- **frp**：需要在内网机器上跑 frpc，配置文件写端口映射，每个服务都要单独配。Red Arrow 走的是 SOCKS5 动态转发，不需要预先指定目标，连上就能访问内网任意地址。
- **ZeroTier / Tailscale**：组虚拟局域网方案，很优雅，但需要内网机器安装客户端，且有些网络环境下打洞不成功需要中继。
- **Red Arrow**：只需要一台能 SSH 的跳板机，不需要内网机器装额外东西，手机端一个 App 搞定。最轻量。

**本质区别：Red Arrow 用的是 SSH 动态端口转发（`ssh -D`），这是最经典、最可靠、最不需要额外组件的内网穿透方式。**

## 技术细节（给感兴趣的同学）

- **语言**：Kotlin
- **SSH 库**：[mwiede/jsch](https://github.com/mwiede/jsch) 0.2.18（社区维护的 JSch 分支）
- **代理实现**：自己写的 SOCKS5（RFC 1928/1929）和 HTTP CONNECT 代理，不依赖第三方库
- **架构**：Foreground Service + Coroutines + StateFlow，响应式 UI
- **最低版本**：Android 8.0（API 26）

代码完全开源，MIT 那种"随便用"的精神（实际是 CC BY-NC-SA 4.0，非商业自由使用）。

## 下载和使用

**GitHub 仓库**：[github.com/JoursBleu/red-arrow](https://github.com/JoursBleu/red-arrow)

**APK 下载**：[v0.1.0-beta](https://github.com/JoursBleu/red-arrow/releases/tag/v0.1.0-beta)

**项目主页**：[joursbleu.github.io/red-arrow](https://joursbleu.github.io/red-arrow/)

使用步骤：

1. 安装 APK
2. 填入你的 SSH 跳板机地址、端口、用户名
3. 选择密码或密钥认证（可以在 App 内直接生成密钥对并发送公钥到服务器）
4. 点击「连接」
5. 把需要访问内网的应用代理设为 `127.0.0.1:1080`（SOCKS5）或 `127.0.0.1:8080`（HTTP）

> **提示**：为了让隧道在后台稳定运行，记得在手机设置里关闭 Red Arrow 的电池优化（省电策略设为"无限制"）。
